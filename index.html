My Work

Python Blackjack

My take on Blackjack, built in Python and run on the command-line. For this version, I used basic Object-Oriented principles to divide responsibilities between a Player class, Dealer Class and Deck class. This project makes use of "if-then" statements, while-loops, looped-exception-handling, and allows up to 4 players.

Java Blackjack

An improved version of my Python Blackjack game, written in Java and making use of design patterns, abstract classes and interfaces in order to allow for flexibility and additional class creation in the future without code reuse. For example, I used the design principle of programming to an interface to allow GameLogic class to accept any class that implements Deck in its constructor, which would allow me to create a brand new Deckobject with totally different cards, should the need arise, without needing to touch GameLogic's code itself.

Also worth mentioning, abstract class Player uses the Template Pattern to leave makeTurnChoice() abstract, requiring subclasses DealerPlayer, HumanPlayer and any future subclasses to define makeTurnChoice()'s behavior specific to their purpose, while still allowing Player subclasses to to share the majority of their code, saving me time and effort.

Stanford Lagunitas SQL Answers

My answers to Stanford Lagunita's DB5 SQL course and Stanford's DB11 Constraints and Triggers course. Throughout these exercises I use subqueries, aggregate functions, trigger creation, relation-modifying statements, and joins (natural, cross, inner and outer) to solve these problems.

BeatBox MVC Redesign

My redesign of the Beat-Box guided project from Head-First Java. The original design for this project included all behavior and attributes in one BeatBox class, whereas my redesign follows Model-View-Controller architecture. My redesign also uses the Observer Pattern to allow BeatView to notify BeatController (which also implements a BeatObserver interface) when the GUI has been interacted with.
